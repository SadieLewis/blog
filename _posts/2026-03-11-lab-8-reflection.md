***Lab 8 Reflection***

I actually had a bit of fun with this lab, especially when I got to make it my own. I don't think there was anything in the tutorial that really messed with me in all honesty. I do think that the scaffolding step was particularly straightforward, though I don't think I encountered many issues in the first place. That said, it did take me a bit longer to wrap my head around the migration stuff with the database. What took me the longest was cloning a new repository on [Github][github] with the code. I ended up duplicating the original code into the repository and making a separate zip file for the tutorial portion. When I did create my new website, Pokedex, I mainly faced challenges converting the database from the old values to the new ones that describe pokemon attributes rather than movie details. I ended up trying a few things after facing errors like adding new types to the database and renaming all the values. However, what got it to work was deleting the database, running the project, then updating the database. That seemed to be what got it to work correctly.

I can't speak from experience with Jekyll (see introduction post), but using Razor framework for building websites wasn't too bad. I feel like I will be able to do better with t the more I use it, but the most challenging part about it was working with the database within the web page. It caused me some struggle, but I came to understand it after a few attempts to try things and make it work. That said, having had that experience I feel much more confident about the possibility of building another Razor page in the future. I will still struggle at some points, but I feel equipped enough to do it!
